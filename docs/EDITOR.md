# 桌宠编辑器使用说明

主要内容：说明 `editor/` 里的桌宠编辑器怎么打开、怎么保存，以及每个页签怎么用。

## 怎么打开

从桌宠状态栏图标右键菜单里点“编辑器”，或者直接运行：

```bash
python editor/main.py
```

如果你平时通过项目自带的虚拟环境运行，也可以用：

```bash
./.venv/bin/python editor/main.py
```

## 怎么保存

多数页签的保存入口是快捷键：

```text
Ctrl+S
```

注意：`Ctrl+S` 保存的是**当前正在看的页签**，不是一次性保存所有页签。

**没有自动保存，没有自动保存，没有自动保存。** 

例如：

1. 切到“互动区域”。
2. 修改鼠标互动区域。
3. 按 `Ctrl+S`。
4. 只会保存 `config/interaction_map.json`。

保存成功后，编辑器会弹出提示。大多数配置需要重新启动桌宠才会生效。

有两个例外：

- “素材配制”主要靠“输出到目标目录”按钮写入素材文件。
- “动作素材”是只读预览页，不需要保存。

## 页签总览

| 页签 | 主要用途 | 写入位置 | 保存方式 |
| --- | --- | --- | --- |
| 素材配制 | 新建素材目录、拖入 PNG、整理帧序列 | `assets/animations/` | “输出到目标目录”，部分删除会立即生效 |
| 动作素材 | 查看素材目录、预览动作动画 | 无 | 只读 |
| 动作注册 | 注册动作 id、显示名、动作类型 | `config/modes.json` | `Ctrl+S` |
| 互动区域 | 配置鼠标按下、点击、拖动不同区域时触发什么 | `config/interaction_map.json` | `Ctrl+S` |
| 自动行为 | 配置默认动作、自动待机、启动退出、随机单次动画 | `config/action_settings.json` | `Ctrl+S` |
| 插件开关 | 启用或禁用插件 | `config/plugin_loader.json` | `Ctrl+S` |

部分页签保存时会先把旧文件备份到：

```text
.vpet_editor_backups/
```

备份文件名里会带时间戳。

## 素材配制

“素材配制”用来把零散 PNG 整理成项目需要的标准动画目录。

它对应的目录结构是：

```text
assets/animations/<动作id>/<状态>/<阶段>/<变体>/<图层>/*.png
```

例如：

```text
assets/animations/touch_head/normal/loop/01/main/_000_125.png
```

### 左侧目录树

左侧是素材目录树。层级从上到下是：

```text
动作 -> 状态 -> 阶段 -> 变体 -> 图层
```

选中不同层级时，下方按钮会启用对应操作：

| 选中层级 | 可做什么 |
| --- | --- |
| 动作 | 新建状态 |
| 状态 | 新建阶段 |
| 阶段 | 新建变体 |
| 变体 | 新建图层 |
| 任意层级 | 删除选中 |

“新建动作...”会在 `assets/animations/` 下创建一个新的动作目录。动作 ID 只允许小写英文、数字、下划线。

可选状态：

```text
happy
normal
poor_condition
ill
any
```

可选阶段：

```text
loop
start
end
single
```

可选图层：

```text
main
back
front
```

### 右侧帧序列

选中左侧的“图层”后，右侧会显示这个图层里的帧序列。

你可以把 PNG 图片直接拖到编辑器窗口里。拖入后，它们会追加到帧列表末尾。

右侧可以调整：

| 项目 | 说明 |
| --- | --- |
| 前缀 | 输出文件名前缀，可以为空 |
| 起始序号 | 新拖入帧从哪个编号开始 |
| 默认延时 | 新拖入帧默认每帧多少毫秒 |
| 帧序列 | 可以上移、下移、删除选中帧 |

输出文件名会变成这种格式：

```text
<前缀>_<序号>_<延时>.png
```

如果前缀为空，会是：

```text
_000_125.png
```

### 输出到目标目录

拖入 PNG 并确认顺序、延时后，点“输出到目标目录”。

这个按钮会把新图片复制到当前选中的图层目录里。

如果目标文件名已经存在，会弹出选择：

| 选项 | 说明 |
| --- | --- |
| 覆盖 | 替换已有文件 |
| 跳过 | 不写这个文件 |
| 自动改名 | 自动加后缀，避免覆盖 |

注意：“素材配制”不是靠 `Ctrl+S` 输出图片。它靠“输出到目标目录”。

### 删除注意事项

左侧“删除选中”会删除选中的目录及其所有内容。

右侧帧列表里的删除会删除帧行；如果那一帧已经在磁盘上，也会删除对应 PNG。但图层目录至少会保留一张 PNG。如果想删除整个图层，请在左侧目录树删除图层。

## 动作素材

“动作素材”是只读查看页，用来确认 `assets/animations/` 里有哪些动作素材，以及素材是否能正常预览。

### 怎么看

左侧是素材目录树：

```text
动作 -> 状态 -> 阶段 -> 变体 -> 图层
```

点击不同层级，右侧会显示不同信息：

| 点击层级 | 右侧会显示 |
| --- | --- |
| 动作 | 尝试预览这个动作 |
| 状态 | 预览这个状态下的动作 |
| 阶段 | 预览这个阶段的第一组可播放变体 |
| 变体 | 预览这个变体 |
| 图层 | 显示图层、帧数、图片尺寸 |

如果动作没有注册，左侧动作名后面会显示“未注册”。

### 刷新素材

如果你在编辑器外部改了素材目录，点“刷新素材”重新扫描。

这个页签不写配置文件，也不需要 `Ctrl+S`。

## 动作注册

“动作注册”用来编辑 `config/modes.json`。

只有注册过的动作，才会出现在菜单、自动行为、互动区域等地方。

### 表格字段

| 字段 | 说明 |
| --- | --- |
| 动作 ID | 必须对应 `assets/animations/<动作id>` |
| 标题 | 菜单或显示用的中文名 |
| 类型 | `loop`、`phased`、`single` |

动作 ID 只允许：

```text
小写英文、数字、下划线
```

### 动作类型

| 类型 | 需要的素材阶段 | 适合什么 |
| --- | --- | --- |
| `loop` | `loop` | 一直循环播放的待机动作 |
| `phased` | `start`、`loop`、`end` | 有开始、持续、结束的动作，比如摸头、睡觉、跳舞 |
| `single` | `single` | 播一次就结束的动作，比如启动、退出、升级 |

### 怎么操作

| 按钮 | 作用 |
| --- | --- |
| 添加 | 新增一行动作 |
| 删除 | 删除选中行 |
| 上移 | 把选中动作往上挪 |
| 下移 | 把选中动作往下挪 |

改完后按 `Ctrl+S` 保存。

保存时会检查：

- 动作 ID 不能为空。
- 动作 ID 不能重复。
- 动作 ID 格式必须合法。
- `assets/animations/` 里必须有对应素材。
- 动作类型必须和素材阶段匹配。
- `action_settings.json` 里引用的动作仍然有效。

如果保存失败，按弹窗提示修正后再保存。

## 互动区域

“互动区域”用来可视化编辑 `config/interaction_map.json`。

左边是桌宠预览图和网格。右边是网格、默认行为、区域列表和选中区域的详细配置。

### 网格

当前默认是 7 行、7 列。行列编号都从 0 开始。

例如：

```text
r0 c0
```

表示第 0 行、第 0 列。

### 默认行为

默认行为表示：如果某个格子没有被任何区域覆盖，或者区域里的某个手势选择“继承默认”，就使用这里的配置。

默认行为有三项：

| 行为 | 什么时候触发 |
| --- | --- |
| 按下 | 鼠标左键按下时 |
| 点击 | 按下后没有拖动，松开时 |
| 拖动 | 按下后移动鼠标时 |

### 区域

一个区域是一块矩形格子范围：

```text
起始行 -> 结束行
起始列 -> 结束列
```

例如：

```text
起始行 0
结束行 1
起始列 2
结束列 4
```

表示覆盖第 0 到 1 行、第 2 到 4 列。

点击左侧预览里的已有色块，会选中对应区域。点击空白格后再点“添加”，会在那个格子上新增一个区域。

右侧“区域”列表可以选中已有区域。“删除”会删除当前选中的区域。

### 行为类型

每个区域可以分别配置按下、点击、拖动。

| 类型 | 说明 |
| --- | --- |
| 继承默认 | 这个区域不单独配置，使用上方“默认行为” |
| 无动作 | 明确什么都不做 |
| 拖动窗口 | 拖动桌宠窗口 |
| 按住动作 | 按住时播放某个 `phased` 动作，松开后恢复 |
| 切换动作 | 切换到某个 `loop` 或 `phased` 动作 |

常见用法：

| 想做的事 | 推荐配置 |
| --- | --- |
| 摸头 | `按下` -> `按住动作` -> `touch_head` |
| 捏脸 | `按下` -> `按住动作` -> `pinch` |
| 提起并拖动桌宠 | `按下` -> `按住动作`，`拖动` -> `拖动窗口` |
| 点击某块区域切换动作 | `点击` -> `切换动作` |

### 重叠规则

互动区域不允许重叠。

如果两个区域覆盖了同一个格子，保存时会报错，例如：

```text
格子 r0 c2 同时被 touch_head_handle 和 overlap_test 覆盖。interaction_map 不允许区域重叠。
```

同一个区域里同时配置 `按下` 和 `拖动` 不算重叠。比如提起区域可以同时有：

```text
按下 -> 按住动作
拖动 -> 拖动窗口
```

### 怎么确认一个格子最终会触发什么

把鼠标移到左侧预览的格子上。左下方会显示这个格子的区域名，以及按下、点击、拖动分别会触发什么。

改完后按 `Ctrl+S` 保存。

## 自动行为

“自动行为”用来编辑 `config/action_settings.json`。

它控制桌宠启动后默认播放什么、空闲时随机切换什么、启动退出播放什么、偶尔插入什么单次动画。

### 基础设置

| 项目 | 说明 |
| --- | --- |
| 默认动作 | 桌宠启动后进入的默认循环动作 |
| 按住互动 | 没有被互动区域单独覆盖时，默认按住桌宠触发的动作 |

“默认动作”只能选择当前默认状态下能播放的 `loop` 或 `phased` 动作。

“按住互动”只能选择 `phased` 动作。

### 空闲自动切换

空闲自动切换表示：桌宠待着不动一段时间后，随机切换到某个待机动作。

| 项目 | 说明 |
| --- | --- |
| 最小间隔 | 最短多久切换一次 |
| 最大间隔 | 最久多久切换一次 |
| 候选动作 | 勾选空闲时允许随机切换的动作 |

候选动作只能是 `loop` 或 `phased`。

### 单次插入

单次插入表示：桌宠待机时偶尔播放一次短动画，播完再回到当前动作。

| 项目 | 说明 |
| --- | --- |
| 最小间隔 | 最短多久插入一次 |
| 最大间隔 | 最久多久插入一次 |
| 候选动作 | 勾选允许随机插入的单次动作 |

候选动作只能是 `single`。

### 启动 / 关机

这里配置桌宠启动和退出时可以随机播放哪些 `single` 动作。

| 项目 | 说明 |
| --- | --- |
| 启动动作 | 启动时随机选择 |
| 关机动作 | 退出时随机选择 |

改完后按 `Ctrl+S` 保存。

保存时会检查最小间隔不能大于最大间隔。

## 插件开关

“插件开关”用来编辑 `config/plugin_loader.json`。

这个页签会扫描 `plugins/` 目录，并列出可用插件。

### 表格字段

| 字段 | 说明 |
| --- | --- |
| 启用 | 勾选表示启用这个插件 |
| 插件 ID | 插件的内部 id |
| 名称 | 插件在菜单里显示的名字 |
| 来源 | 插件文件路径 |

插件可以是：

```text
plugins/<插件id>.py
plugins/<插件id>/plugin.py
```

如果 `plugin_loader.json` 里启用了某个插件，但文件已经不存在，这一行会显示为缺失插件。

### 怎么操作

1. 勾选要启用的插件。
2. 取消勾选要禁用的插件。
3. 如果你刚新增或删除了插件文件，点“刷新插件”。
4. 按 `Ctrl+S` 保存。
5. 重启桌宠。

## 改完后怎么确认生效

1. 在对应页签完成修改。
2. 用对应方式保存：`Ctrl+S` 或“输出到目标目录”。
3. 关闭桌宠。
4. 重新启动桌宠。
5. 测试对应行为。

如果是互动区域，可以打开开发模式看网格是否和预期一致。

开发模式配置在：

```text
config/window_settings.json
```

把 `dev_mode` 改成 `true` 后重启桌宠，会显示互动网格和播放调试信息。

## 常见问题

### 按了 Ctrl+S 但没变化

先确认保存时所在的页签是不是你刚刚修改的页签。编辑器只保存当前页签。

然后重启桌宠。大多数配置不会热更新。

### 素材配制按 Ctrl+S 没反应

“素材配制”主要靠“输出到目标目录”写入图片文件。拖入 PNG 后，要点“输出到目标目录”。

### 动作下拉框里找不到想要的动作

先确认动作已经写进 `config/modes.json`，也就是“动作注册”页签里能看到它。

还要确认动作类型是否匹配：

| 配置项 | 允许的动作类型 |
| --- | --- |
| 按住动作 | `phased` |
| 切换动作 | `loop`、`phased` |
| 启动 / 退出 / 单次插入 | `single` |

### 动作注册保存失败

常见原因：

- 动作 ID 拼错了。
- `assets/animations/<动作id>` 目录不存在。
- 动作类型和素材阶段不匹配。
- 自动行为里还引用了已经删除或改名的动作。

先去“动作素材”页签看素材是否存在，再回到“动作注册”修正。

### 保存互动区域时报重叠

说明两个区域盖住了同一个格子。选中报错里提到的区域，调整起始行、结束行、起始列、结束列，让它们不要覆盖同一个格子。

### 改了插件开关但插件状态没变

插件开关需要保存并重启桌宠后才会生效。

### 不知道某个格子最终会触发什么

在“互动区域”页签里，把鼠标移到左侧预览的格子上。左下方会显示这个格子的区域名，以及按下、点击、拖动分别会触发什么。
